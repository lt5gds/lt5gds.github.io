    Rangkuman Dokumentasi Libgdx

    LibGDX merupakan sebuah framework atau API tambahn yang digunakan sebagai membuat game dengan berbagai macam platfom

    File Handling

    Libgdx's Files (code) class dapat melakukan:

    • membaca file
    • menulis ke file
    • menduplikat file
    • memindahkan file
    • menghapus file
    • List files and directories
    • menchek file ada atau tidak

    Arsitektur

    Aplikasi merupakan point utama untuk macam-macam project Libgdx. Sebuah Aplikasi menentukan target platform dan graphics backend yang akan digunakan. Interface aplikasi juga menyediakan metode yang terbaik untuk mengakses graphics, audio, input dan file I/O module yang umum. Interface aplikasi juga memberian akses ke Logging module yang umum yang bekerja pada semua cross-platform. Sekarang ini, Libgdx mendukung dua backend untuk aplikasi dekstop (lwjgl dan jogl) dan satu backend untuk Android. Berikut merupakan diagram alir (aplication Life-cycle) pada aplikasi Libgdx.



    Class Audio

    menggunakan class audio

    Audio audio=gdx.audio

    audio dihentikan secara otomatis saat metode ApplicationListener.pause() dipanggil, dan dilanjutkan secara otomatis saat metode ApplicationListener.resume()dipanggil.

    fungsi

    • newAudioDevice(int samplingRate, boolean isMono)
    • newAudioRecorder(int samplingRate, boolean isMono)
    • newMusic(FileHandle file) -> membuat musik yang berputar saat dimainkan
    • newSound(FileHandle fileHandle) -> membuat musik yang digunakan sebagai effek dalam game

    Graphics

    Viewports
    Saat berhadapan dengan layar yang berbeda, seringkali perlu diputuskan untuk menentukan strategi tertentu bagaimana ukuran dan rasio aspek layar yang berbeda harus ditangani. Camera and Stage mendukung strategi viewport yang berbeda, misalnya saat melakukan pemotretan melalui Camera.project (vec, viewportX, viewportY, viewportWidth, viewportHeight).

    2D Graphic

    • SpriteBatch, TextureRegions and Sprites

      SpriteBatch digunakan untuk menngambar sebuah bentuk layaknya canvas pada html dan javascript, sedangkan texture digunakan untuk memunculkan sebuah file gambar dan textureregion digunakan untuk membuat kotak tempat textur digambar sedankan Sprite untuk textureregion dan batch.

    • 2D Animation

      digunakan untuk membuat ilusi gerakan dengan gambar yang satis dengan mengubah gambar pada setipa frame dengan class Animation

    • Orthographic Camera

      class OrthographicCamera digunakan sebagai dasar dari 2D yaitu paralel projeksi, method yang ada dalam class ini yaitu rotate(float angle);
      setToOrtho() translate(float x, float y) translate(vector2 vec) update()

    Input Handling

    Event Handling adalah menangkap setiap tindakan user sebagai input seperti swip, touch up, touch down di tombol ata berati saat tombol diklik, keadan yang dapat ditangkap antara lain, keydown() //saat tombol keyboard di tekan
    keyup() //saat melepas tombol keyboard
    keyTyped() //saat mengetik
    touchDown() //saat moouse diklik ditekan maka mengahasilkan kordinat
    touchUp() //saat mouse klik diangkat
    touchDragged() //saat mouse mendrag sesuatu discreen
    mouseMoved() //saat pinter mengunjungi suatu area hanya terjadi di dekstop
    scrolled() //saat discroll
